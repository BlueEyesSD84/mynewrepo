1. A relational database includes tables containing rows and columns. For example, a typical business order entry database would include a table that describes a customer with columns for name, address, phone number and so forth.

- NoSQL databases feature dynamic schema, and allow you to use what’s known as “unstructured data.” This means you can build your application without having to first define the schema. In a relational database(SQL/MySQL), you are required to define your schema before adding data to the database. Not needing a predefined schema makes NoSQL databases much easier to update as data and requirements change

2. C.R.U.D.(Create,Read,Update,Delete) are the four basic RDBMS operations. The following are definitions and examples of sql statements.
Create(Insert function) to insert data into a database. Create: INSERT INTO menu VALUES (1, 'croissant', 1, '2020-12-16');

Read(Select function) to select information from a database. READ: IF OBJECT_ID('cusp_CustomerRead') IS NOT NULL BEGIN DROP PROC cusp_CustomerRead END GO CREATE PROC cusp_CustomerRead @CustomerID int AS BEGIN SELECT CustomerID, FirstName, LastName, Email, PhoneNumber FROM Customer WHERE (CustomerID = @CustomerID) END GO

Update(edit function) is used to Edit information on a database. UPDATE: UPDATE menu SET item_name = 'chocolate croissant', price = 2.5 WHERE item_id = 1;

Delete(Delete function) used to remove data from a database. DELETE: DELETE FROM menu WHERE item_name='bread';
SQL Scan Operation: Is used to Scan the entire index or table. Example: SELECT * FROM Employee.
